01 / PROTECT THE CALLER

Circuit Breaker

When a downstream service starts failing, stop hammering it. A circuit breaker trips open after repeated failures, short-circuiting new calls immediately so the caller fails fast instead of piling up timeouts — then cautiously tests the water in a half-open state before fully closing again.

02 / RECOVER FROM BLIPS

Retry with Backoff

Transient failures — a dropped packet, a momentary GC pause — often resolve themselves. Retrying immediately just adds to the pile-up, so each attempt waits longer than the last, giving the dependency room to recover.

03 / DON'T WAIT FOREVER

Timeout

A slow dependency without a timeout is worse than a dead one — it ties up threads and memory while everyone waits. A firm deadline turns "hung" into a fast, well-understood failure.

Where this comes from

Aggressive, tuned-per-dependency timeouts freed up request threads fast, which is what made fallback logic and circuit breakers effective in the first place.

04 / CONTAIN THE BLAST RADIUS

Bulkhead Isolation

Named after a ship's watertight compartments: give each dependency its own resource pool (threads, connections) so one overwhelmed call path can't starve every other feature of the resources it needs.

Separate thread pools per dependency meant a stalled "recommendations" call couldn't exhaust the threads checkout needed to keep charging cards.

05 / SHED LOAD ON PURPOSE

Rate Limiting & Load Shedding

A token bucket refills at a steady rate; every request spends one token. When the bucket's empty, new requests are rejected outright — a controlled, cheap failure that keeps the service alive for everyone else instead of collapsing under load.
